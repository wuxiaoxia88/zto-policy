# 测试与调试规范

**项目**: 中通面单返利测算工具
**日期**: 2026-05-08

---

## 1. 测试目标

本项目的测试优先级:

1. 公式正确。
2. 口径清楚。
3. 中文 UI 不溢出、不乱码。
4. 导入导出稳定。
5. 非专业用户能看懂结论。

计算错误是最高优先级缺陷，必须立即修复。

---

## 2. 自动化测试

### 2.1 单元测试

覆盖模块:

| 模块 | 必测内容 |
|---|---|
| `date.ts` | 月份天数、闰年、手动覆盖天数 |
| `rebate.ts` | 档位匹配、基础返利、分层补贴、单票返利 |
| `penalty.ts` | 指标缺口、同比负增长、两种处罚取其一 |
| `cost.ts` | 简易成本、重量段成本、缺失成本 |
| `customer.ts` | 单客户、新增客户、存量客户增量、大客户降价 |
| `increment.ts` | 增量增利、增量减利、叠加客户 |
| `recommendation.ts` | 建议做、谨慎做、不建议做、数据不足 |

### 2.2 必须通过的业务用例

| 用例 | 预期 |
|---|---|
| 2026-05 月份 | 自动识别 31 天 |
| 2026-06 月份 | 自动识别 30 天 |
| 业务量指标 8000，实际 7500 | 缺口 500 票/天 |
| 未达标处罚 0.35 元/票 | 处罚按 500 × 0.35 × 天数 |
| 同比负增长处罚 0.40 元/票 | 与 0.35 处罚二选一，不叠加 |
| 利润测算案例一 | 最终净影响 `-38.8 元/天` |
| 利润测算案例二 | 最终净影响 `+105.2 元/天` |

## 3. UI 测试

### 3.1 桌面端

断点:

1. 1366 × 768。
2. 1440 × 900。
3. 1600 × 1000。
4. 1920 × 1080。

检查项:

1. 三栏布局不重叠。
2. 中文表头不截断。
3. 核心金额清晰可读。
4. 图表 Tooltip 中文显示正常。
5. 滑块拖动后图表同步刷新。
6. 客户叠加表操作正常。

### 3.2 移动端

断点:

1. 390 × 844。
2. 430 × 932。

检查项:

1. 单列布局顺序正确。
2. 参数区可收起。
3. 客户表格转为卡片。
4. 保存图片不裁切、不乱码。
5. 按钮高度不低于 44px。

---

## 4. 导入测试

导入类型:

1. 政策模板。
2. 网点参数。
3. 客户列表。
4. 成本模板。
5. 业务量指标。

每类文件必须测试:

| 场景 | 预期 |
|---|---|
| 字段完整 | 可导入 |
| 缺少必填字段 | 显示缺字段 |
| 数值格式错误 | 显示无法导入 |
| 档位区间重叠 | 阻止导入 |
| 与现有模板同名 | 提示覆盖冲突 |
| 用户选择追加 | 不覆盖原数据 |
| 用户选择替换 | 覆盖当前数据 |

---

## 5. 导出测试

### 5.1 数据导出

格式:

1. CSV。
2. XLSX。
3. JSON。

检查项:

1. 中文表头正常。
2. 金额、票量、百分比单位正确。
3. 方案对比数据完整。
4. 隐私开关生效。

### 5.2 图片导出

尺寸:

1. 手机分享图 1080 × 1920。
2. 长图报告 1080 × 2400 或自适应高度。
3. 会议横图 1920 × 1080。
4. 单卡片图 1200 × 900。

检查项:

1. 中文不乱码。
2. 图表不空白。
3. 数据不被裁切。
4. 隐藏客户名称、报价、成本后导出内容正确。
