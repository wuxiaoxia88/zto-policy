# 使用说明

**项目**: 中通面单返利测算工具
**适用对象**: 网点老板、店长、业务负责人、财务/数据人员、管区政策负责人
**说明**: 本工具为非官方辅助测算工具，最终结果以管区政策和实际结算为准。

---

## 1. 工具能解决什么问题

本工具用于回答三个问题:

1. 当前网点按新政策预计能拿多少面单返利。
2. 新增或调整客户后，网点整体是增利、持平还是减利。
3. 为了跳档或减少业务量处罚，某个客户是否值得做，报价底线是多少。

---

## 2. 两种使用视角

### 2.1 管区视角

适合政策宣讲和沟通。

主要看:

1. 政策档位。
2. 当前档和下一档。
3. 距离下一档还差多少票。
4. 跳档后全量返利变化。
5. 增量增利和增量减利案例。

导出内容默认隐藏客户敏感字段和网点内部毛利。

### 2.2 网点视角

适合经营测算和报价决策。

主要看:

1. 月度总返利。
2. 单票返利。
3. 业务量处罚风险。
4. 客户叠加后毛利变化。
5. 保本报价和决策建议。

---

## 3. 使用前需要准备的数据

### 3.1 政策参数

| 参数 | 说明 |
|---|---|
| 档位 | 不同票量区间 |
| 基数比例 | 对应档位的浮动基数比例 |
| 基础补贴 | 基础返利标准 |
| 分层补贴 | 各档位分层补贴 |
| 分层基数 | 由省区给每个网点单独制定 |

### 3.2 网点参数

| 参数 | 说明 |
|---|---|
| 测算月份 | 系统自动识别当月天数 |
| 政策有效票量 | 用于返利计算的票量 |
| 业务量考核票量 | 用于业务量处罚计算的全量票量 |
| 去年同期业务量 | 用于同比负增长判断 |

### 3.3 客户参数

| 参数 | 说明 |
|---|---|
| 客户名称 | 可匿名 |
| 客户类型 | 新增、存量增量、降价增量、大客户 |
| 日均票量 | 客户预计每天发件量 |
| 报价 | 给客户的收件价格 |
| 成本 | 单票成本或重量段成本 |
| 退件率 | 服装等品类可填写 |
| 退件收益 | 退件带来的额外收入 |

### 3.4 成本参数

成本可以简单填写，也可以导入模板。

简单模式:

```text
统一单票成本 = X 元/票
```

精细模式:

```text
0-0.3kg
0.3-0.5kg
0.5-1kg
1-2kg
2-3kg
3-5kg
```

## 4. 基本使用流程

```text
选择视角 -> 选择月份 -> 选择或导入政策模板 -> 填写网点参数 -> 添加客户 -> 查看结论 -> 导出分享
```

当前 `v0.6.0` 已开放两种视角：网点视角用于经营测算，管区视角用于政策宣讲。管区视角会隐藏客户敏感字段和网点内部毛利，只展示公开政策、返利变化、处罚风险和典型案例。

### 4.1 当前网点测算

1. 选择测算月份。
2. 填写政策有效票量。
3. 填写业务量考核票量。
4. 填写分层基数。
5. 查看月度总返利、单票返利和档位。

### 4.1.1 管区政策宣讲

1. 点击顶部“管区视角”。
2. 填写或调整测算月份、当前日均票量、业务量指标和分层基数。
3. 查看当前档位、距离下一档、返利提升和处罚风险。
4. 使用“典型网点案例”和“宣讲话术”进行会议讲解。
5. 导出数据时仅导出公开宣讲总览和公开案例。

### 4.2 跳档模拟

1. 拖动日均票量滑块。
2. 查看当前档位和下一档。
3. 关注“还差 X 票/天进入下一档”。
4. 查看跳档后月度返利和单票返利变化。

### 4.3 客户叠加测算

1. 点击“追加客户”。
2. 填写客户票量、报价、成本、退件率。
3. 可继续追加多个客户。
4. 每个客户可启用、停用、删除。
5. 查看最终净影响和决策建议。

### 4.4 业务量处罚测算

1. 填写季度/月度业务量指标。
2. 填写实际业务量。
3. 填写去年同期业务量。
4. 系统判断按未达标处罚还是同比负增长处罚。
5. 两种处罚只取其一，不叠加。

### 4.5 JSON 参数导入

当前 MVP 支持导入 JSON 文件，字段示例:

```json
{
  "siteInputs": {
    "siteName": "无锡某网点",
    "month": "2026-06",
    "policyEffectiveDailyVolume": 7500,
    "businessDailyVolume": 7500,
    "layerBaseDailyVolume": 4500,
    "businessTargetDailyVolume": 8000,
    "sameMonthLastYearDailyVolume": 7600
  },
  "customers": [
    {
      "id": "customer-a",
      "name": "服装客户 A",
      "enabled": true,
      "policyEffectiveDailyVolume": 500,
      "businessDailyVolume": 500,
      "pricePerTicket": 2.5,
      "costPerTicket": 2.18,
      "returnRate": 0.5,
      "returnIncomePerTicket": 1.2
    }
  ]
}
```

## 5. 如何理解结果

### 5.1 月度总返利

表示当前测算月份预计可获得的面单返利总额。

### 5.2 单票返利

分为两种口径:

1. 政策有效单票返利。
2. 全量单票返利。

如果业务量口径和政策有效口径差异较大，应重点看全量单票返利。

### 5.3 最终净影响

最终净影响是决策核心，综合考虑:

1. 存量收益变化。
2. 新增客户自身利润。
3. 业务量处罚变化。
4. 退件收益。
5. 成本变化。

### 5.4 决策建议

系统会输出:

1. 建议做。
2. 谨慎做。
3. 不建议做。
4. 数据不足。

如果成本数据未填写，系统只输出返利和处罚结论，不输出完整毛利判断。

## 6. 导出与分享

### 6.1 数据导出

网点视角点击“导出数据”会生成 XLSX 文件，包含:

1. 测算总览。
2. 客户明细。
3. 增量拆解。

管区视角点击“导出数据”会生成公开宣讲 XLSX，只包含政策宣讲总览和公开案例，不包含客户敏感字段或内部毛利。

后续版本会继续补充 CSV、JSON、参数模板和隐私开关。

### 6.2 图片导出

当前版本点击“保存图片”会保存当前工作台图片，适合内部沟通和快速截图分享。

后续版本将继续增强:

1. 手机分享图。
2. 长图报告。
3. 会议横图。
4. 单卡片图。

导出前可选择隐藏:

1. 客户名称。
2. 客户报价。
3. 成本明细。
4. 敏感备注。

---

## 7. 注意事项

1. 票量增加不一定代表利润增加。
2. 新客户自身亏损时，必须看是否能带来足够的存量收益提升或处罚减少。
3. 成本数据越完整，决策建议越可靠。
4. 政策模板必须以最新管区政策为准。
5. 本工具只做辅助测算，不作为最终结算依据。
