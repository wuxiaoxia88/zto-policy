## 1. 背景与问题

江苏/苏南发件政策从原来多套补丁式政策，调整为“浮动基数 + 分层补贴”的统一政策框架。政策目标是拉齐大小网点成本差距，鼓励网点主动开发客户，尤其是让中小网点能够通过新增客户、跳档、全量返利提升整体竞争力。

但网点实际决策中存在几个难点:

1. 各省区、各管理区、不同时间段的档位区间、基数比例、基础补贴、分层补贴可能不同，不能写死。
2. 返利不是简单的“新增票量 × 单票补贴”，跳档后会影响全量返利、分层补贴和单票成本。
3. 网点老板真正关心的不是政策公式本身，而是“这个客户能不能做”“降价后还赚不赚”“为了跳档要不要接边际客户”。
4. 单客户、存量客户增量、新增客户、整体网点冲量、业务量考核等口径混在一起，靠手算容易误判。
5. 需要用直观图表、滑块、动态对比，把跳档前后的变化展示清楚。

本 PRD 将旧版“返利展示工具”升级为“返利 + 成本 + 客户报价 + 决策建议”的综合测算工具。

## 2. 产品目标

### 2.1 一句话目标

让网点经营者输入当前网点数据和客户方案后，能快速看到返利、成本、单票毛利、跳档收益、业务量风险的前后变化，并找到当前情况下的最优报价和最优冲量方案。

### 2.2 核心目标

1. **政策可配置**: 省区档位、档位区间、基数比例、基础补贴、分层补贴、分层基数、业务量指标都可自定义，并支持模板保存、复制、套用、修改。
2. **跳档可视化**: 通过滑块模拟日均票量变化，实时看到档位、总返利、单票返利、分层补贴、整体成本的变化。
3. **成本精细化**: 支持网点固定成本、半固定成本、边际成本、揽收成本、业务员激励、税差、退件收益等参数。
4. **客户决策化**: 支持单客户、新增客户、存量客户增量、大客户降价、整体网点冲量等场景测算。
5. **方案对比化**: 支持保存多个方案，对比新增票量、报价、返利变化、毛利变化、单票成本变化和推荐结论。
6. **建议自动化**: 输出“建议做/谨慎做/不建议做”的决策参考，并说明关键原因和风险边界。

## 3. 用户与使用场景

### 3.1 用户角色

| 角色 | 主要诉求 | 使用频率 |
|---|---|---:|
| 网点老板 | 判断客户能不能做、报价能降到多少、是否要冲量跳档 | 高频 |
| 店长/业务负责人 | 开发客户前测算底价，和业务员分解任务 | 高频 |
| 财务/数据人员 | 校验政策参数、核算返利、导出方案 | 中频 |
| 网格长/政策负责人 | 宣讲政策，验证网点提交的测算逻辑 | 中频 |

### 3.2 关键场景

1. **当前网点返利测算**
   输入当前日均票量、分层基数、政策模板，看到本月预计返利、单票返利、档位、分层补贴。

2. **滑块跳档模拟**
   拖动日均票量滑块，从当前票量向上/向下模拟，看到跳档临界点、返利总额、单票返利、全量成本变化。

3. **新增客户测算**
   输入新增客户票量、报价、重量结构、边际成本，计算新增客户带来的返利增量、成本增量、毛利增量。

4. **存量客户增量测算**
   例如客户当前 500 票/天、价格 2.50 元；若做到 1000 票/天，要求价格降到 2.20 元。工具需要对比前后收入、成本、返利、毛利。

5. **整体网点冲量测算**
   输入多个客户或一个目标日均票量，计算是否值得为了跳档接边际客户，以及最少需要新增多少票才能达到下一个收益更优点。

6. **多方案比较**
   对比“保守报价”“成本价拿量”“降价换量”“只接高毛利客户”等方案，选择综合收益最优方案。

## 4. 政策规则模型

### 4.1 苏南默认政策模板

来自 `苏南标准.md`，作为系统内置默认模板之一，用户可复制后调整。

| 档位 | 日均区间 | 基数比例 | 基础补贴 | 分层补贴 |
|---:|---|---:|---:|---:|
| 1 | 0-500 | 10% | 1.50 | 1.10 |
| 2 | 501-1000 | 75% | 1.50 | 0.80 |
| 3 | 1001-2000 | 70% | 1.50 | 0.75 |
| 4 | 2001-5000 | 45% | 1.50 | 0.40 |
| 5 | 5001-10000 | 35% | 1.50 | 0.28 |
| 6 | 10001-30000 | 30% | 1.50 | 0.26 |
| 7 | 30001-50000 | 20% | 1.50 | 0.12 |
| 8 | 50001-100000 | 15% | 1.50 | 0.04 |
| 9 | 100001 以上 | 12% | 1.50 | 0.00 |

### 4.2 核心计算口径

系统需要区分两类票量:

| 口径 | 说明 | 用途 |
|---|---|---|
| 政策有效票量 `Qe` | 0-5kg 标准件，剔除散件、一口价、逆向件、刷单件等不享受返利票 | 判定返利档位、计算基础返利和分层补贴 |
| 业务量考核票量 `Qt` | 全量业务量，含 5kg 以上等不享受返利但计入业务量的票 | 业务量指标、达标/未达标风险测算 |

待实现时，界面要允许用户选择“暂时用同一个票量口径”或“分别录入政策有效票量与业务量考核票量”。

### 4.3 基础返利公式

变量:

| 字段 | 含义 |
|---|---|
| `Qe` | 政策有效日均票量 |
| `Qt` | 业务量考核日均票量 |
| `M` | 测算月份，如 2026-05 |
| `D` | 当月天数，由系统根据 `M` 自动识别，如 2026-05 为 31 天，2026-06 为 30 天 |
| `r` | 当前档位基数比例 |
| `p` | 当前档位基础补贴标准，苏南默认 1.50 元/票 |
| `s` | 当前档位分层补贴 |
| `B` | 分层基数，由省区给每个网点单独制定 |

公式:

```text
当前档位 = 按 Qe 匹配政策档位
基础返利日票量 = Qe × (1 - r)
基础返利日金额 = 基础返利日票量 × p

分层补贴日票量 = max(Qe - B, 0)
分层补贴日金额 = 分层补贴日票量 × s

单日总返利 = 基础返利日金额 + 分层补贴日金额
月度总返利 = 单日总返利 × D
政策有效单票返利 = 单日总返利 ÷ Qe
全量单票返利 = 单日总返利 ÷ Qt
```

### 4.4 月份与天数口径

系统不要求用户手动判断每月天数。用户选择测算月份后，系统自动计算 `D`:

```text
D = daysInMonth(M)
```

示例:

| 测算月份 | 自动天数 |
|---|---:|
| 2026-05 | 31 |
| 2026-06 | 30 |
| 2026-02 | 28 |

月总返利、月收入、月成本、月处罚、月毛利均默认使用该月份天数计算。用户可在高级设置中手动覆盖天数，用于特殊结算周期或非自然月场景。

### 4.5 叠加方案公式

系统不能只支持“方案前/方案后”两个状态，必须支持继续追加客户，形成可编辑的叠加测算链。

基础结构:

```text
基础状态 S0 = 当前网点状态
客户变动 C1 = 新增/调整第 1 个客户
客户变动 C2 = 新增/调整第 2 个客户
...
叠加状态 Sn = S0 + C1 + C2 + ... + Cn
```

每个客户变动都可以独立调整:

1. 客户名称。
2. 客户类型。
3. 当前票量。
4. 目标票量。
5. 报价。
6. 重量段结构。
7. 是否开票。
8. 退件率。
9. 单票成本或成本模板。
10. 是否计入政策有效票量。
11. 是否计入业务量考核票量。

对任意叠加状态，系统都要重新计算:

```text
返利增量 = 叠加后月度总返利 - 基础状态月度总返利
收入增量 = 叠加后客户收入 - 基础状态客户收入
成本增量 = 叠加后客户成本 - 基础状态客户成本
其他收益增量 = 税差收益 + 退件收益 + 其他自定义收益
处罚/风险增量 = 叠加后业务量处罚 - 基础状态业务量处罚

月度毛利增量 = 收入增量 + 返利增量 + 其他收益增量 - 成本增量 - 处罚/风险增量
新增票单票净贡献 = 月度毛利增量 ÷ 新增月票量
```

界面上应允许用户任意调整每个客户件量，图表和结论实时刷新；也要展示每追加一个客户后对档位、返利、处罚和毛利的边际影响。

### 4.6 报价底线公式

用于回答“这个客户最低能报多少钱”:

```text
新增票可分摊返利 = 返利增量 ÷ 新增月票量
新增票其他收益 = 其他收益增量 ÷ 新增月票量
新增票边际成本 = 成本增量 ÷ 新增月票量

保本报价 = 新增票边际成本 - 新增票可分摊返利 - 新增票其他收益
目标毛利报价 = 保本报价 + 目标单票毛利
```

说明: 保本报价不是建议直接对外报价，而是经营底线。最终报价仍需考虑客户质量、同行报价真实性、服务成本、账期、开票要求、跨区合规等因素。

### 4.7 业务量处罚风险模型

业务量处罚会影响增量业务决策，必须进入毛利测算。业务量统计口径包含所有快件:

1. 线上快递。
2. 逆向快递。
3. 自发自签。
4. 一客一价。
5. 5kg 以上快件。
6. 其他计入业务量的全量快件。

变量:

| 字段 | 含义 |
|---|---|
| `TargetQt` | 省区按季度制定的业务量指标，折算为日均票量 |
| `ActualQt` | 实际业务量日均票量 |
| `YoYQt` | 去年同期同月业务量日均票量 |
| `Shortfall` | 业务量缺口，`max(TargetQt - ActualQt, 0)` |
| `PenaltyA` | 未达指标处罚 |
| `PenaltyB` | 未达指标且同比负增长处罚 |

默认规则:

```text
Shortfall = max(TargetQt - ActualQt, 0)

若 Shortfall = 0:
  业务量处罚 = 0

若 Shortfall > 0 且 ActualQt >= YoYQt:
  PenaltyA = Shortfall × 未达指标处罚单价 × D
  业务量处罚 = PenaltyA

若 Shortfall > 0 且 ActualQt < YoYQt:
  PenaltyB = Shortfall × 负增长处罚单价 × D
  业务量处罚 = PenaltyB
```

两条处罚不叠加，满足哪条就按哪条处罚；如果后续政策文件存在多个可选处罚结果，则取政策规定结果，不做重复叠加。

示例:

```text
2026-06 业务量指标 = 8000 票/天
2026-06 实际业务量 = 7500 票/天
缺口 = 500 票/天

若未同比负增长:
  日处罚 = 500 × 0.35 = 175 元/天
  月处罚 = 175 × 30 = 5250 元

若相对去年同期 2025-06 负增长:
  日处罚 = 500 × 0.40 = 200 元/天
  月处罚 = 200 × 30 = 6000 元
```

在客户叠加测算中，新增客户可能减少业务量缺口、降低处罚，系统应把“避免处罚金额”计入方案收益的一部分。

### 4.8 退件收益模型

部分大客户或特定品类会带来额外退件收益。例如服装品类可能存在 50% 左右平均退货率，退件会给发件网点带来额外收入。该因素会影响“是否值得做”的判断。

变量:

| 字段 | 含义 |
|---|---|
| `ReturnRate` | 退件率 |
| `ReturnIncomePerTicket` | 单票退件收入 |
| `ReturnCostPerTicket` | 单票退件额外成本 |

公式:

```text
月退件量 = 客户月发件量 × ReturnRate
月退件净收益 = 月退件量 × (ReturnIncomePerTicket - ReturnCostPerTicket)
```

退件收益默认作为高级参数，可在客户模板或客户测算中单独开启。系统输出决策建议时，必须说明是否依赖退件收益；如果方案只有计入退件收益才盈利，应提示风险。

### 4.9 发件成本模型

系统需要引入另一套发件成本系统，用于更准确测算客户实际发件成本。前期 MVP 允许用户手动输入、简单录入、导入或留空；后期与独立成本测算系统对接。

默认重量段:

| 重量段 |
|---|
| 0-0.3kg |
| 0.3-0.5kg |
| 0.5-1kg |
| 1-2kg |
| 2-3kg |
| 3-5kg |
| 5kg 以上 |

成本计算:

```text
客户单票发件成本 = Σ(重量段占比 × 该重量段成本)
客户月发件成本 = 客户月发件量 × 客户单票发件成本
```

成本来源优先级:

1. 客户单独录入的成本。
2. 客户选择的成本模板。
3. 全局默认成本模板。
4. 空值，系统只做返利测算并提示“成本未填，毛利结论不完整”。

### 4.10 存量收益与新增件自身收益拆分

`利润测算案例一.md` 和 `利润测算案例二.md` 提供了一个非常重要的测算口径: 增量业务可能带来跳档、操作成本下降、浮动基数补贴提升，从而让原有存量件增利；但新增件本身可能因为报价过低而亏损，也可能在低利润下仍为正贡献。最终要比较“存量收益提升”和“新增件自身收益”合计后的净影响。

系统必须把整体影响拆成两部分:

```text
存量收益提升 = 存量件因跳档/降本/补贴变化带来的收益增加
新增件自身收益 = 新增件收入 - 新增件系统成本 - 新增件操作成本 + 新增件可归属补贴
最终净影响 = 存量收益提升 + 新增件自身收益
```

结论判定:

| 结果 | 判定 |
|---|---|
| 存量收益提升 + 新增件自身收益 > 0 | 增量增利 |
| 存量收益提升 + 新增件自身收益 = 0 | 增量持平 |
| 存量收益提升 + 新增件自身收益 < 0 | 增量减利 |

该拆分必须进入网点视角的决策建议。尤其当新增件自身亏损时，系统不能直接判定“不建议做”，而要继续判断它带来的存量收益提升、业务量处罚减少、退件收益等是否足以覆盖亏损；当新增件自身低利润但为正贡献时，也要解释其与存量收益共同形成“增量增利”的原因。

## 5. 数据与模板需求

### 5.1 模板类型

| 模板 | 用途 | 是否必需 |
|---|---|---|
| 政策模板 | 省区/管理区档位、基数比例、补贴标准、分层补贴 | 必需 |
| 网点模板 | 网点名称、当前票量、分层基数、业务量指标、默认月份 | 必需 |
| 成本模板 | 面单、中转、揽收、操作、客服、业务员补贴、税率、重量段成本等 | 可选 |
| 客户模板 | 客户类型、重量结构、报价、开票、退件率、揽收方式 | 可选 |
| 方案模板 | 常用测算方案，如“新增客户”“存量增量”“降价换量” | 必需 |

### 5.2 模板能力

1. 新建模板。
2. 复制模板。
3. 修改模板。
4. 删除模板。
5. 恢复系统默认模板。
6. 一键套用模板到当前测算。
7. 导入/导出 JSON。
8. 本地自动保存。

### 5.2.1 参数录入方式

所有关键参数都必须提供两种设置方式:

| 方式 | 目标用户 | 适用场景 |
|---|---|---|
| 手工输入 | 网点老板、店长、临时测算用户 | 少量参数、现场演示、快速试算 |
| 文件导入 | 管区、财务、数据人员 | 批量导入政策、客户、成本、业务量指标 |

必须支持该原则的参数范围:

1. 政策档位参数。
2. 网点基础参数。
3. 分层基数。
4. 业务量指标和去年同期业务量。
5. 客户列表和客户报价。
6. 客户重量段结构。
7. 发件成本模板。
8. 退件率和退件收益参数。
9. 方案列表。

导入格式 MVP 至少支持 CSV/XLSX；系统应提供下载模板，避免用户不知道该填哪些字段。导入后必须进入预览确认页，显示成功行、异常行、缺失字段和覆盖策略，不能静默覆盖已有数据。

### 5.3 政策模板字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 模板名称 | 文本 | 如“苏南 2026-05 默认” |
| 生效日期 | 日期 | 用于历史版本区分 |
| 省区/管理区 | 文本 | 如江苏/苏南/无锡 |
| 档位列表 | 表格 | 每档包含下限、上限、基数比例、基础补贴、分层补贴 |
| 是否封顶 | 布尔 | 默认不封顶 |
| 政策有效范围 | 多选 | 默认 0-5kg 标准件 |
| 剔除范围 | 多选 | 散件、一口价、逆向件、刷单件等 |

### 5.4 成本模板字段

成本模板不是必填项。用户可以选择导入、简单录入或完全不填。不填时，系统仍可完成返利、跳档和处罚测算，但毛利、保本报价、决策建议必须标记为“成本数据不足”。

| 成本项 | 输入方式 | 说明 |
|---|---|---|
| 面单成本 | 元/票 | 可按客户或全局配置 |
| 重量段发件成本 | 元/票表格 | 支持 0-0.3、0.3-0.5、0.5-1、1-2、2-3、3-5、5kg 以上 |
| 中转费 | 元/票或分重量段 | 支持一区/二区/三区、均重、折扣；可先并入重量段发件成本 |
| 揽收成本 | 元/票 | 自有揽收可为 0 或低值，外包按票配置 |
| 操作成本 | 元/票 | 如卸货、扫描、装车等 |
| 业务员激励 | 元/票或元/月 | 新客户可能需要额外补贴业务员 |
| 客服/售后成本 | 元/票或元/月 | 可选 |
| 固定成本 | 元/月 | 房租、人员、水电、车辆固定支出 |
| 半固定成本 | 阶梯 | 如超过某票量需加车/加人 |
| 开票税点 | 百分比 | 区分开票/不开票客户 |
| 退件收益 | 退件率 × 单票收益 | 可选，用于会议中提到的退件派费收益 |
| 罚款风险 | 元/票 | 业务量未达标、负增长等 |

导入格式 MVP 支持 CSV/XLSX 二选一，字段至少包含:

```text
目的省 / 重量段 / 单票成本
```

如果用户导入 `发件成本测算.md` 类似的省份 × 重量段矩阵，系统需能转换为成本模板；客户测算时按目的省和重量段占比计算加权成本。

### 5.5 业务量考核模板字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 考核季度 | 年-季度 | 省区按季度制定 |
| 适用月份 | 多选月份 | 如 2026-04、2026-05、2026-06 |
| 业务量指标 | 票/天 | 日均指标 |
| 去年同期业务量 | 票/天 | 用于判断是否负增长 |
| 未达标处罚单价 | 元/票 | 默认示例 0.35 |
| 负增长处罚单价 | 元/票 | 默认示例 0.40 |
| 处罚叠加规则 | 枚举 | 默认不叠加，满足哪条按哪条 |
| 业务量统计范围 | 多选 | 线上快递、逆向快递、自发自签、一客一价、5kg 以上等 |

## 6. 功能需求

### 6.0 双视角模式

系统必须提供两种视角: 管区视角和网点视角。两种视角共用同一套计算引擎，但默认展示重点、文案、图表和导出物不同。

#### 管区视角

目标: 用于管区向网点展示政策变化，帮助网点理解政策的公平性、合理性、普惠性，最大化政策推广效果。

重点:

1. 展示不同规模网点在新政策下的返利变化。
2. 用直观图表说明“做增量、跳档、全量返利”为什么更有利。
3. 强调政策对中小网点的支持和普惠效果。
4. 帮助网点降低理解门槛，减少“账算不清”的疑虑。
5. 支持生成适合会议、微信群、朋友圈传播的政策讲解图。

默认模块:

1. 政策档位总览。
2. 不同票量区间的返利曲线。
3. 跳档示例。
4. 小网点/中网点/大网点对比。
5. 政策公平性说明。
6. 分享图片导出。

#### 网点视角

目标: 用于网点自己测算不同发件量、客户报价、发件成本带来的财务变化，指导开发客户和精准报价。

重点:

1. 当前网点能拿多少返利。
2. 新增客户是否值得做。
3. 客户降价后是否仍然有利润。
4. 业务量处罚是否会因为增量客户而减少。
5. 保本报价和建议报价区间。
6. 多客户叠加后的整体毛利变化。

默认模块:

1. 当前网点总览。
2. 客户叠加测算。
3. 成本和报价测算。
4. 业务量处罚风险。
5. 方案对比。
6. 决策建议。
7. 数据导出和图片导出。

#### 视角切换要求

1. 顶部提供“管区视角 / 网点视角”分段切换。
2. 切换视角不丢失已录入参数。
3. 管区视角默认隐藏过细成本字段，避免宣传场景复杂化。
4. 网点视角默认展开成本、客户、报价、处罚相关字段。
5. 导出内容跟随当前视角生成不同报告。

### 6.1 首页总览

展示当前测算的核心结论:

1. 当前政策模板。
2. 测算月份和自动天数。
3. 当前网点日均政策有效票量 `Qe`。
4. 当前业务量考核票量 `Qt`。
5. 当前档位。
6. 分层基数 `B`。
7. 月度总返利。
8. 业务量处罚风险。
9. 政策有效单票返利。
10. 全量单票返利。
11. 距离下一档还差多少票。
12. 当前方案建议: 建议做/谨慎做/不建议做。

### 6.2 政策参数配置

要求:

1. 档位数量不固定，支持 1-N 档。
2. 每档区间可编辑，区间需校验连续性和重叠。
3. 基数比例、基础补贴、分层补贴都可编辑。
4. 用户修改后，所有图表和测算结果实时刷新。
5. 支持“苏南标准”作为默认模板。
6. 支持保留旧版江苏 12 档模板，用于历史比较或旧项目兼容。

### 6.3 滑块模拟器

核心交互:

1. 用户拖动日均票量滑块。
2. 页面实时更新档位、返利、单票返利、分层补贴、成本、毛利。
3. 滑块上标注当前档、上一档、下一档关键临界点。
4. 进入下一档时，显示跳档提示和变化拆解。
5. 支持“从整月平均”与“从每天票量”两种视角。

滑块视角:

| 视角 | 说明 |
|---|---|
| 月均模式 | 输入整月平均日均票量，按月均回算 |
| 每日模式 | 录入或导入每天票量，自动算月均，并显示日波动对月均的影响 |

### 6.4 跳档前后变化面板

当用户接近档位边界或方案导致跳档时，必须展示:

1. 跳档前档位、跳档后档位。
2. 基数比例变化。
3. 基础返利变化。
4. 分层补贴变化。
5. 月度总返利变化。
6. 单票返利变化。
7. 新增票量贡献。
8. 全量返利带来的额外收益。

图表建议:

1. 瀑布图: 收入增量、返利增量、成本增量、税差、退件收益、最终毛利。
2. 阶梯曲线: 票量变化对应的总返利和单票返利。
3. 差值卡片: 跳档前后关键指标对比。

### 6.5 成本参考工具

分为简易模式和精细模式。

#### 简易模式

输入:

1. 单票边际成本。
2. 固定成本/月。
3. 测算月份，系统自动识别当月天数。
4. 新增客户票量。
5. 客户报价。

输出:

1. 返利增量。
2. 成本增量。
3. 月度毛利增量。
4. 单票净贡献。
5. 保本报价。
6. 若成本未填，输出返利结论并提示毛利不可完整判断。

#### 精细模式

输入:

1. 面单成本。
2. 中转费。
3. 揽收成本。
4. 操作成本。
5. 业务员激励。
6. 客服/售后成本。
7. 固定成本分摊规则。
8. 半固定成本阶梯。
9. 开票税点。
10. 退件率和退件收益。
11. 业务量考核指标和罚款规则。
12. 重量段成本模板或客户单独成本。

输出:

1. 单票变动成本。
2. 单票固定成本摊销。
3. 单票综合成本。
4. 单客户毛利。
5. 整体网点毛利变化。
6. 跳档收益对客户的可分摊金额。
7. 推荐报价区间。
8. 业务量处罚减少或增加的金额。

### 6.6 单客户测算

适用于判断一个客户是否值得做。

输入:

1. 客户名称。
2. 客户类型: 新增客户/存量客户/存量增量/大客户降价。
3. 当前票量。
4. 目标票量。
5. 当前报价。
6. 目标报价。
7. 是否开票。
8. 是否外包揽收。
9. 客户退件率。
10. 客户重量结构。

输出:

1. 客户自身收入变化。
2. 客户自身成本变化。
3. 网点返利变化。
4. 客户贡献毛利。
5. 该客户拉动跳档后的全网点收益。
6. 若把跳档收益全部或部分让给客户，最低报价是多少。

必须支持示例:

```text
现有客户: 500 票/天，价格 2.50 元
目标情况: 1000 票/天以上，价格降到 2.20 元
输出: 前后收入、成本、返利、毛利、单票净贡献、是否建议接受降价
```

### 6.7 整体网点测算

适用于老板从全局判断要不要冲量。

输入:

1. 当前网点政策有效票量。
2. 当前网点业务量考核票量。
3. 目标日均票量。
4. 一个或多个新增客户方案。
5. 当前分层基数。
6. 固定成本与边际成本。

输出:

1. 当前档位与目标档位。
2. 当前月返利与目标月返利。
3. 返利增量。
4. 成本增量。
5. 毛利增量。
6. 单票成本变化。
7. 距离业务量指标的差距。
8. 是否存在处罚风险。
9. 系统建议的最优新增票量区间。

### 6.8 新增客户组合测算

支持添加多个潜在客户，形成组合方案。

每个客户包含:

1. 日均票量。
2. 报价。
3. 单票成本。
4. 是否开票。
5. 是否外包。
6. 可拿下概率。
7. 客户质量评分。

系统输出:

1. 哪些客户组合能跳到下一档。
2. 哪些客户组合毛利最高。
3. 哪些客户组合风险最低。
4. 哪些客户“单看不赚钱，但带动跳档后整体赚钱”。
5. 哪些客户虽然毛利一般，但能减少业务量处罚，值得优先开发。

### 6.9 叠加客户测算

用户可以在一个方案中持续追加客户，并任意调整每个客户的件量、报价、成本、退件率和是否计入不同票量口径。每一次调整都要实时刷新总结果。

交互要求:

1. 客户列表以可编辑表格或卡片展示。
2. 每个客户可启用/停用，用于观察该客户对整体结果的影响。
3. 支持拖动客户票量滑块，快速模拟客户从 0 到目标票量的变化。
4. 支持复制客户，快速创建相似客户。
5. 支持查看“加入该客户前/加入该客户后”的边际变化。

叠加结果必须展示:

1. 叠加后的政策有效票量。
2. 叠加后的业务量考核票量。
3. 档位变化。
4. 返利变化。
5. 业务量处罚变化。
6. 成本变化。
7. 退件收益变化。
8. 总毛利变化。

### 6.10 方案对比

用户可保存多个方案:

1. 当前方案。
2. 新增 100 票方案。
3. 新增 500 票方案。
4. 大客户降价方案。
5. 成本价拿量方案。
6. 目标跳档方案。
7. 多客户叠加方案。

对比字段:

| 指标 | 说明 |
|---|---|
| 目标日均票量 | 方案后票量 |
| 档位变化 | 前档 -> 后档 |
| 月度返利 | 总返利 |
| 返利增量 | 相比当前 |
| 业务量处罚 | 当前处罚与方案后处罚 |
| 避免处罚收益 | 方案帮助减少的罚款 |
| 月度毛利 | 含成本后 |
| 毛利增量 | 相比当前 |
| 单票返利 | 政策有效口径/全量口径 |
| 单票综合成本 | 含固定成本摊销 |
| 保本报价 | 新增票最低经营底线 |
| 数据完整度 | 成本、退件、税差、去年同期等是否完整 |
| 风险提示 | 未达标、负增长、客户质量、退件依赖等 |

### 6.11 决策建议

系统根据规则输出建议，建议必须引入数据参考并完整说明理由，不能只给一句“建议做”。

| 建议 | 条件示例 |
|---|---|
| 建议做 | 月度毛利增量 > 0，且无重大业务量/跨区/客户质量风险 |
| 谨慎做 | 毛利接近 0，但能帮助达标或跳档，需要关注报价和成本 |
| 不建议做 | 毛利明显为负，且无法带来跳档、达标、税差、退件等收益 |
| 需要人工确认 | 缺少关键成本、客户报价真实性存疑、跨区合规不明确 |

建议文案必须包含:

1. 结论。
2. 主要原因。
3. 关键敏感参数。
4. 可接受的最低报价。
5. 若要做，建议控制条件。
6. 数据依据列表。
7. 缺失数据提醒。

数据依据至少包括:

1. 当前月度返利与方案后月度返利。
2. 返利增量。
3. 业务量处罚变化。
4. 收入增量。
5. 成本增量。
6. 退件净收益。
7. 月度毛利增量。
8. 单票净贡献。
9. 保本报价。

示例:

```text
建议做。该客户从 500 票/天提升到 1000 票/天后，虽然单票报价从 2.50 降到 2.20，但带动网点月返利增加 18,600 元，扣除新增边际成本后，月度毛利仍增加 7,200 元。建议报价底线不低于 2.12 元，并关注开票要求和揽收成本。
```

### 6.12 数据导出与图片分享

系统必须提供测算数据导出和保存图片能力，方便传播、汇报、对账和沟通。

#### 数据导出

MVP 支持:

2. 导出方案对比表。
3. 导出客户叠加明细。
4. 导出政策参数和成本参数 JSON，方便下次复用。

导出字段至少包含:

1. 测算月份。
2. 当前政策模板。
3. 当前网点票量。
4. 分层基数。
5. 业务量指标和处罚。
6. 客户列表。
7. 客户报价。
8. 成本参数。
9. 返利结果。
10. 毛利结果。
11. 决策建议。

#### 图片导出

MVP 支持:

1. 保存当前结果卡为图片。
2. 保存管区视角政策讲解图。
3. 保存网点视角方案对比图。
4. 保存单客户报价测算图。

图片要求:

1. 默认适配手机查看和微信群传播。
2. 不包含复杂小字，关键结论必须清晰。
3. 标注“非官方测算，最终以管区结算为准”。
4. 可显示生成时间、政策模板、测算月份。

#### 不同视角的导出差异

| 视角 | 导出重点 |
|---|---|
| 管区视角 | 政策解释、档位曲线、跳档收益、公平性/普惠性展示 |

### 6.13 增量增利/增量减利分析

系统必须提供“增量增利/增量减利”专项分析，用来解释新增业务到底有没有让网点整体更赚钱。

输入:

1. 原有存量件量。
2. 多轮新增件量。
3. 每轮新增后的档位。
4. 每轮新增后的收件价格。
5. 系统成本。
6. 操作成本。
7. 浮动基数补贴。
8. 分层补贴。

输出:

1. 新增件自身单票利润。
2. 新增件自身总利润。
3. 原有存量件因降本带来的收益。
4. 原有存量件因补贴变化带来的收益。
5. 前序新增件因后续跳档带来的收益变化。
6. 分层补贴下降带来的损失。
7. 最终净影响。
8. 结论: 增量增利/增量持平/增量减利。

可视化:

1. 使用瀑布图展示“存量增利 + 前序新增增利 - 最后一批新增亏损 = 最终净影响”。
2. 使用表格展示每一批客户/件量的边际贡献。
3. 当最终净影响为负时，用红色提示“票量增加但利润下降”。

案例要求:

`利润测算案例二.md` 中的结果应能被系统表达为:

```text
存量收益提升 = 94.8 元/天
第二次新增 200 件自身利润 = 10.4 元/天
最终净影响 = 105.2 元/天
结论 = 增量增利
```

`利润测算案例一.md` 中的结果应能被系统表达为:

```text
存量收益提升 = 167.2 元/天
第三次新增 1000 件自身亏损 = -206 元/天
最终净影响 = -38.8 元/天
结论 = 增量减利
```

## 7. 可视化需求

可视化目标不是炫技，而是让非专业人士一眼看懂“为什么这个客户值得做或不值得做”。所有图表必须围绕经营问题设计，优先展示结论、差额、临界点和风险。

### 7.1 必备图表

1. **返利-票量阶梯曲线**
   横轴为日均票量，纵轴为月度返利，标注档位分界点。

2. **单票返利曲线**
   展示政策有效单票返利和全量单票返利。

3. **跳档前后瀑布图**
   拆解收入增量、返利增量、成本增量、税差、退件收益、最终毛利。

4. **单票成本/返利对比曲线**
   展示单票综合成本、单票边际成本、单票返利、单票毛利。

5. **方案对比柱状图**
   多方案对比月度毛利、返利增量、风险。

6. **敏感性热力图**
   横轴报价，纵轴新增票量，颜色表示月度毛利或是否建议做。

7. **客户贡献堆叠图**
   展示单客户收入、成本、返利分摊、其他收益的构成。

8. **业务量处罚风险图**
   展示业务量指标、当前业务量、叠加方案后业务量和缺口罚款变化。

### 7.2 动态可视化

1. 滑块拖动时图表实时更新。
2. 经过档位临界点时，曲线高亮并显示跳档收益。
3. 方案保存后，图表显示多条方案线。
4. 支持一键切换“整月平均/每天明细”“网点整体/单客户/新增客户”。
5. 支持鼠标悬停查看具体票量下的返利、成本、毛利。

### 7.3 易懂性与稳定性要求

1. 图表曲线视觉上要平滑，但政策档位跳变必须保留清晰的阶梯和断点提示，不能为了平滑隐藏真实政策变化。
2. 普通用户不需要理解公式，也能通过颜色和数字看懂“多赚/少赚/罚款减少/风险增加”。
3. 图表默认只显示当前决策相关范围，例如当前档位上下 1-2 档，避免全量范围过大导致看不清。
4. 关键数字要直接标在图上，如“还差 500 票跳档”“少罚 5250 元”“保本报价 2.12 元”。
5. 技术实现优先使用成熟稳定方案，如 ECharts、Chart.js、Canvas/SVG 混合渲染、requestAnimationFrame 节流、响应式降级。
6. 动画只用于辅助理解，不能影响读数稳定性；低性能设备上应自动降级为静态图表。
7. 所有图表在无成本数据、无退件数据、无去年同期数据时都要能稳定显示，并明确标注数据缺失。

## 9. 数据校验与边界

### 9.1 输入校验

1. 档位区间不能重叠。
2. 档位区间不能断档，除非用户明确允许。
3. 基数比例范围 0%-100%。
4. 补贴金额不能小于 0。
5. 分层基数不能小于 0。
6. 政策有效票量不能大于业务量考核票量，除非用户选择忽略校验。
7. 客户目标票量不能小于 0。
8. 报价不能小于 0。
9. 成本不能小于 0。

### 9.2 异常提示

1. 未达到分层基数时，明确提示“只享受基础返利，不享受分层补贴”。
2. 票量下降导致降档时，明确提示返利下降风险。
3. 新增客户导致跳档但单客户自身亏损时，提示“该客户需从整体网点维度判断”。
4. 业务量未达标时，提示潜在处罚。
5. 客户报价低于保本报价时，提示不建议做。

## 11. 验收标准

### 11.1 政策测算验收

用苏南模板验证:

1. `Qe=1500, B=1000`
   - 档位: 第 3 档
   - 基础返利: `1500 × (1 - 70%) × 1.5 = 675` 元/日
   - 分层补贴: `(1500 - 1000) × 0.75 = 375` 元/日
   - 单日总返利: `1050` 元
   - 单票返利: `0.70` 元/票

2. `Qe=2500, B=1000`
   - 档位: 第 4 档
   - 基础返利: `2500 × (1 - 45%) × 1.5 = 2062.50` 元/日
   - 分层补贴: `(2500 - 1000) × 0.40 = 600` 元/日
   - 单日总返利: `2662.50` 元
   - 单票返利: `1.065` 元/票

3. `Qe=8000, B=11000`
   - 档位: 第 5 档
   - 基础返利: `8000 × (1 - 35%) × 1.5 = 7800` 元/日
   - 分层补贴: `0` 元/日
   - 明确提示未达到分层基数

4. `Qe=35000, B=11000`
   - 档位: 第 7 档
   - 基础返利: `35000 × (1 - 20%) × 1.5 = 42000` 元/日
   - 分层补贴: `(35000 - 11000) × 0.12 = 2880` 元/日

### 11.2 客户测算验收

必须支持以下测试:

```text
网点当前 Qe=8400
客户当前 500 票/天，报价 2.50
客户目标 1000 票/天，报价 2.20
输入单票边际成本后，系统输出:
1. 前后客户收入变化
2. 前后网点返利变化
3. 成本增量
4. 月度毛利增量
5. 保本报价
6. 是否建议接受降价
```

### 11.3 月份天数验收

1. 用户选择 `2026-05`，系统自动按 31 天计算月度返利、成本、处罚和毛利。
2. 用户选择 `2026-06`，系统自动按 30 天计算月度返利、成本、处罚和毛利。
3. 用户切换月份后，所有月度金额实时刷新。

### 11.4 业务量处罚验收

```text
测算月份: 2026-06
自动天数: 30
业务量指标 TargetQt = 8000 票/天
实际业务量 ActualQt = 7500 票/天
缺口 Shortfall = 500 票/天
未达标处罚单价 = 0.35 元/票
负增长处罚单价 = 0.40 元/票
```

验收结果:

1. 若 `ActualQt >= YoYQt`，月处罚 = `500 × 0.35 × 30 = 5250` 元。
2. 若 `ActualQt < YoYQt`，月处罚 = `500 × 0.40 × 30 = 6000` 元。
3. 两条处罚不叠加，不能出现 `11250` 元这类叠加结果。
4. 当用户追加客户使实际业务量达到 8000 票/天，处罚应变为 0，并把减少的处罚计入方案收益。

### 11.5 多客户叠加验收

1. 用户可在当前网点基础上追加客户 A、客户 B、客户 C。
2. 用户调整任意客户的件量后，整体票量、档位、返利、处罚、成本、毛利同步刷新。
3. 用户停用客户 B 后，系统只按客户 A 和客户 C 重新计算。
4. 每个客户都能展示自己的边际贡献。

### 11.6 增量增利/增量减利案例验收

#### 案例二: 增量增利

以 `利润测算案例二.md` 为样例:

```text
原业务量 = 300 件
第一次新增 = 200 件
第二次新增 = 200 件

存量收益提升 = 94.8 元/天
第二次新增自身利润 = 10.4 元/天
最终净影响 = 105.2 元/天
```

验收结果:

1. 系统能识别该方案为“增量增利”。
2. 系统能展示存量收益提升和第二次新增件自身利润。
3. 系统能说明增利原因: 虽然新增件价格下降到 2.1 元，但跳档带动存量件降本和补贴提升，最终整体利润增加。
4. 瀑布图能展示 `+94.8 + 10.4 = +105.2`。

#### 案例一: 增量减利

以 `利润测算案例一.md` 为样例:

```text
原业务量 = 300 件
第一次新增 = 200 件
第二次新增 = 200 件
第三次新增 = 1000 件

存量收益提升 = 167.2 元/天
第三次新增自身亏损 = -206 元/天
最终净影响 = -38.8 元/天
```

验收结果:

1. 系统能识别该方案不是简单的“票量增加”，而是“增量减利”。
2. 系统能分别展示存量收益提升和新增件自身亏损。
3. 系统能说明亏损原因: 第三次新增件单票利润为负，且亏损额超过存量收益提升。
4. 瀑布图能展示 `+167.2 - 206 = -38.8`。

### 11.7 成本与退件验收

1. 用户不填成本时，系统仍输出返利和处罚结果，但毛利结论标记为数据不足。
2. 用户录入重量段成本和客户重量结构后，系统计算加权单票发件成本。
3. 服装客户设置 50% 退件率时，系统计算退件净收益，并在决策建议里说明该收益对结论的影响。

### 11.8 可视化验收

1. 拖动滑块时，核心数字和图表实时变化。
2. 跳档点必须在曲线上清晰标记。
3. 至少支持 3 个方案横向对比。
4. 单票口径必须能在“政策有效单票”和“全量单票”之间切换。
5. 简易模式用户 1 分钟内能得到结论。
6. 曲线视觉平滑、标注清楚，非专业用户能看懂关键结论。
7. 低性能设备上图表不崩溃，必要时自动关闭复杂动画。

## 12. 待确认问题

1. 各省区是否都采用“政策有效票量判档”，还是部分地区用全量业务量判档。
2. 分层基数 `B` 的下发口径是否按网点固定，原则上一年内不大调，但是否允许月度微调。
3. `业务量考核标准.md` 当前项目目录暂未找到；需补齐后校验业务量处罚单价、业务量统计口径、季度指标和同比口径。
4. 一口价、散件、逆向件、5kg 以上件是否只计业务量，不计返利，需按实际结算口径最终确认。
5. 税差、退件派费、客户开票要求是否作为默认字段展示，还是放入高级选项。
6. 成本模板是否需要按重量段、流向、均重进一步展开。
7. `发件成本测算.md` 中 `5+` 列的数值口径需确认，是 5kg 以上附加成本、续重成本还是完整单票成本。
