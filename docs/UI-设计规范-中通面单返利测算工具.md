## 1. 审核结论

### 1.1 原 UI 规范可保留的部分

Gemini 版本中有几处方向是可用的:

1. 使用中通深蓝 `#0072BC` 和浅蓝 `#00AEEF` 作为品牌主色。
2. 提到“管区视角 / 网点视角”的顶部切换。
3. 提到客户表格、滑块模拟、瀑布图、方案对比等关键模块。
4. 强调全中文界面。
5. 图表选择 ECharts 的方向正确。

### 1.2 主要问题

1. **没有形成真正的设计系统**
   原文更多是模块描述，缺少布局栅格、组件状态、字段密度、导入流程、导出规范和移动端规则。

2. **没有充分区分双视角**
   管区视角应该偏政策宣讲和传播，网点视角应该偏经营测算和决策。原文只是提到切换，没有定义两套信息层级和导出内容。

3. **图表规范不够业务化**
   面单返利是阶梯政策，不能用“平滑曲线”弱化跳档。视觉可以平顺，但数据表达必须保留阶梯、断点和临界点。

4. **参数设置没有产品化**
   所有参数都要支持手工输入和导入。原文没有定义统一的“手工/导入/模板/预览确认/异常处理”流程。

5. **没有覆盖增量增利/增量减利解释**
   本项目最核心的决策不是“票量增加”，而是解释存量收益提升、新增件自身利润、处罚变化、退件收益共同作用后的净影响。

6. **导出传播能力不足**
   工具要用于会议、微信群、网点沟通，必须从 UI 层面定义图片导出、数据导出和匿名分享。

7. **视觉偏通用仪表盘**
   需要更像“经营决策工作台”，信息密度要比营销页高，但比财务系统更容易理解。

## 2. 设计目标

### 2.1 产品定位

本系统不是展示型大屏，也不是传统财务软件，而是一个中文经营决策工作台。

核心体验:

```text
选政策 -> 填/导入参数 -> 拖票量 -> 加客户 -> 看图表解释 -> 得出建议 -> 导出分享
```

### 2.2 设计原则

| 原则 | 说明 |
|---|---|
| 全中文 | 页面、按钮、表头、提示、导出图片全部中文 |
| 一眼看结论 | 关键结论必须用大数字和明确标签表达 |
| 公式可追溯 | 每个结果都能展开查看计算依据 |
| 阶梯要真实 | 政策跳档必须用阶梯、断点、临界线表达 |
| 参数易设置 | 所有参数支持手工输入和文件导入 |
| 双视角清晰 | 管区视角重宣讲，网点视角重测算 |
| 可传播 | 输出适合微信群、会议、汇报的图片和表格 |
| 稳定克制 | 动效辅助理解，不制造干扰 |

## 4. 信息架构

### 4.1 全局框架

桌面端采用“三栏工作台”。

```text
顶部栏
  品牌 / 视角切换 / 模板 / 月份 / 导入 / 导出 / 保存

左栏 参数区
  当前网点
  政策参数
  业务量指标
  成本参数
  高级设置

中栏 决策区
  核心结果卡
  票量滑块
  客户叠加表
  决策建议

右栏 解释区
  返利阶梯图
  瀑布图
  处罚风险
  方案对比
```

推荐宽度:

```text
左栏 320px
中栏 minmax(560px, 1fr)
右栏 420px
页面最大宽度 1600px
```

移动端采用单列分区:

```text
核心结论
视角切换
票量模拟
客户叠加
图表解释
参数设置
导出分享
```

### 4.2 双视角模式

#### 管区视角

用途: 政策宣讲、会议讲解、网点沟通、微信群传播。

设计重点:

1. 少参数，多解释。
2. 强调政策公平性、合理性、普惠性。
3. 展示典型网点从低档到高档的收益变化。
4. 强调跳档带来的全量收益。
5. 默认展示“政策讲解图导出”。

首屏模块:

| 区域 | 内容 |
|---|---|
| 核心结论 | 本政策如何影响不同规模网点 |
| 档位图 | 苏南 9 档或当前政策模板 |
| 跳档说明 | 距离下一档、跳档后增加多少 |
| 典型案例 | 增量增利、增量减利对比 |
| 导出 | 保存政策讲解图 |

#### 网点视角

用途: 网点自测、客户开发、报价决策、成本收益判断。

设计重点:

1. 参数完整。
2. 客户可叠加。
3. 成本可手输或导入。
4. 决策建议必须说明理由。
5. 导出用于内部沟通或给业务员执行。

首屏模块:

| 区域 | 内容 |
|---|---|
| 核心结论 | 月度总返利、毛利增量、处罚风险、保本报价 |
| 客户叠加 | 多客户件量、报价、成本、启用状态 |
| 增量分析 | 存量收益提升、新增件自身收益、最终净影响 |
| 图表解释 | 阶梯图、瀑布图、处罚风险图 |
| 导出 | 保存方案对比图、导出明细 |

## 5. 参数设置 UI

### 5.1 总原则

所有关键参数必须提供两种入口:

```text
手工输入
文件导入
```

不要把导入功能藏在高级设置里。管区、财务、数据人员会大量依赖导入。

### 5.2 参数面板结构

左栏参数区分为 5 组。

| 分组 | 默认状态 | 内容 |
|---|---|---|
| 网点基础 | 展开 | 网点名称、测算月份、有效票量、业务量票量 |
| 政策参数 | 折叠 | 政策模板、档位表、基础补贴、分层补贴 |
| 业务量考核 | 折叠 | 季度指标、去年同期、处罚单价 |
| 成本参数 | 网点视角展开 | 简易成本、重量段成本、固定成本 |
| 导入管理 | 折叠 | 政策、客户、成本、业务量导入 |

### 5.3 手工输入模式

表单要求:

1. 标签在上，输入框在下。
2. 单位固定显示在输入框右侧，例如 `票/天`、`元/票`、`元/月`。
3. 数字输入右对齐。
4. 错误信息紧贴输入框下方。
5. 所有金额和票量采用千分位展示。
6. 输入后实时计算，但复杂表格采用 300ms 防抖。

字段示例:

```text
政策有效票量
[ 8,113        ] 票/天

分层基数
[ 4,500        ] 票/天

未达标处罚
[ 0.35         ] 元/票
```

### 5.4 文件导入模式

导入采用四步流程:

```text
选择类型 -> 上传文件 -> 预览校验 -> 确认导入
```

导入类型:

| 类型 | 文件格式 | 关键字段 |
|---|---|---|
| 政策模板 | CSV/XLSX | 档位、下限、上限、基数比例、基础补贴、分层补贴 |
| 网点参数 | CSV/XLSX | 网点名称、月份、有效票量、业务量票量、分层基数 |
| 客户列表 | CSV/XLSX | 客户名称、类型、票量、报价、成本、退件率 |
| 成本模板 | CSV/XLSX | 目的省、重量段、单票成本 |
| 业务量指标 | CSV/XLSX | 季度、月份、指标、去年同期、处罚单价 |

预览校验:

| 状态 | UI |
|---|---|
| 成功 | 绿色标签“可导入” |
| 缺字段 | 橙色标签“需补字段” |
| 格式错误 | 红色标签“无法导入” |
| 覆盖冲突 | 蓝色提示“将覆盖当前模板” |

导入后不直接覆盖，必须提供:

```text
替换当前数据
追加到当前数据
另存为新模板
```

### 5.5 参数完整度提示

页面右上角显示数据完整度。

```text
数据完整度 8/11
缺少: 去年同期业务量、退件收益、重量段成本
```

完整度影响决策建议:

| 完整度 | 建议等级 |
|---|---|
| 高 | 可输出明确建议 |
| 中 | 输出建议，但提示缺失项 |
| 低 | 只输出返利和处罚，不输出毛利结论 |

## 6. 核心组件规范

### 6.1 顶部栏

内容:

```text
中通面单返利测算工具
[管区视角 | 网点视角]
政策模板
测算月份
导入
导出数据
保存图片
保存方案
```

要求:

1. 顶部栏固定，滚动时保持可见。
2. 视角切换使用分段控件，不使用下拉框。
3. 导入、导出、保存图片使用图标 + 中文文字。
4. 当前模板和月份始终可见。

### 6.2 核心结果卡

结果卡必须突出“结论”和“变化”。

网点视角主卡:

| 卡片 | 内容 |
|---|---|
| 月度总返利 | 金额大数字，展示基础返利/分层补贴拆分 |
| 最终净影响 | 增利为绿，减利为红 |
| 单票返利 | 政策有效口径/全量口径切换 |
| 业务量处罚 | 当前处罚、方案后处罚、少罚金额 |
| 保本报价 | 新增客户最低报价参考 |

管区视角主卡:

| 卡片 | 内容 |
|---|---|
| 政策覆盖 | 当前政策适用范围 |
| 档位变化 | 当前档、下一档、跳档缺口 |
| 跳档收益 | 跳档后全量收益变化 |
| 普惠说明 | 小网点/中网点/大网点对比 |

### 6.3 票量滑块

票量滑块是核心交互，但不能变成炫技控件。

要求:

1. 滑块上方显示当前票量。
2. 滑块下方显示档位临界点。
3. 接近下一档时显示“还差 X 票/天”。
4. 跨档时弹出气泡说明档位变化。
5. 滑块下方同步显示“当前档 -> 下一档”的关键变化。
6. 移动端提供 `-100`、`+100`、`+500` 快捷步进按钮。

不要使用烟花、夸张光晕等娱乐化动效。政策工具要专业可信。

### 6.4 客户叠加表

表格列:

| 列 | 说明 |
|---|---|
| 启用 | Switch |
| 客户 | 客户名称，可匿名 |
| 类型 | 新增/存量增量/降价/大客户 |
| 票量 | 票/天，可滑块调整 |
| 报价 | 元/票 |
| 成本 | 元/票或成本模板 |
| 退件率 | 百分比 |
| 边际贡献 | 元/月 |
| 结论 | 增利/持平/减利 |
| 操作 | 复制、删除、展开 |

展开行展示:

1. 重量段占比。
2. 开票税点。
3. 退件收益。
4. 是否计入政策有效票量。
5. 是否计入业务量考核票量。
6. 计算明细。

### 6.5 决策建议面板

建议面板固定在中栏核心区域下方。

结构:

```text
结论标签: 建议做 / 谨慎做 / 不建议做 / 数据不足
一句话结论
关键理由 3 条
数据依据
缺失数据
建议控制条件
```

示例:

```text
结论: 谨慎做
该客户自身单票亏损 0.206 元，但能带来存量收益提升 167.2 元/天。
由于新增件自身亏损 206 元/天，高于存量收益提升，最终净影响 -38.8 元/天。
建议除非该客户还能减少业务量处罚或带来退件收益，否则不建议按当前报价接入。
```

### 6.6 方案对比卡

方案卡不宜做大而空的营销卡，要像经营对比表。

每张卡显示:

1. 方案名称。
2. 目标票量。
3. 档位变化。
4. 月度返利。
5. 业务量处罚。
6. 最终净影响。
7. 结论标签。
8. 数据完整度。

自动高亮:

| 规则 | UI |
|---|---|
| 毛利最高 | 绿色边框 |
| 风险最高 | 红色角标 |
| 数据不完整 | 橙色提示 |
| 当前选中 | 中通蓝边框 |

## 7. 图表可视化规范

### 7.1 图表总原则

1. 政策曲线必须是阶梯线，不得用平滑曲线表达政策跳档。
2. 视觉线条可以抗锯齿，但数据形态必须保留断点。
3. 图表必须直接回答经营问题。
4. 图例用中文，不出现英文占位。
5. 所有 Tooltip 必须显示单位。
6. 图表必须支持导出图片。

### 7.2 图表一: 返利阶梯曲线

用途:

展示票量变化如何影响月度返利和单票返利。

图形:

| 元素 | 规范 |
|---|---|
| X 轴 | 日均票量 |
| Y 轴 | 月度返利或单票返利 |
| 线型 | 阶梯线 `step: start/end` |
| 当前点 | 中通蓝实心点 |
| 下一档 | 虚线标记 |
| 档位区间 | 浅蓝底色分段 |

Tooltip:

```text
日均票量: 8,113 票/天
当前档位: 第 5 档
月度返利: 204,663 元
单票返利: 0.814 元/票
距离下一档: 1,888 票/天
```

### 7.3 图表二: 增量增利/减利瀑布图

用途:

解释为什么最终增利或减利。

柱子顺序:

```text
基础状态
存量降本
浮动基数补贴增加
分层补贴变化
新增件自身利润
处罚减少
退件净收益
最终净影响
```

颜色:

| 类型 | 色彩 |
|---|---|
| 起点/终点 | 中通蓝 |
| 正向收益 | 成功绿 |
| 负向损失 | 错误红 |
| 未填/估算 | 灰色虚线 |

案例二应展示:

```text
+94.8 + 10.4 = +105.2
结论: 增量增利
```

案例一应展示:

```text
+167.2 - 206 = -38.8
结论: 增量减利
```

### 7.4 图表三: 业务量处罚风险图

用途:

解释新增客户是否减少处罚风险。

推荐图形:

1. Bullet Chart。
2. 指标线。
3. 当前实际值。
4. 叠加方案后实际值。
5. 缺口罚款标签。

文案:

```text
当前缺口 500 票/天，预计处罚 5,250 元/月
叠加客户后达到 8,000 票/天，处罚降为 0
少罚 5,250 元/月
```

### 7.5 图表四: 成本收益堆叠图

用途:

解释单客户收入、成本、返利分摊、退件收益。

结构:

```text
收入
- 系统成本
- 操作成本
- 业务员激励
+ 返利分摊
+ 退件收益
= 单客户净贡献
```

### 7.6 图表五: 报价敏感性热力图

MVP 可暂缓，但设计预留。

用途:

横轴为客户报价，纵轴为新增票量，颜色表示最终净影响。

颜色:

| 区域 | 含义 |
|---|---|
| 深绿 | 明显增利 |
| 浅绿 | 小幅增利 |
| 灰色 | 持平 |
| 浅红 | 小幅减利 |
| 深红 | 明显减利 |

## 8. 交互状态规范

### 8.1 通用状态

所有关键组件必须具备:

| 状态 | 要求 |
|---|---|
| Loading | 骨架屏或进度条 |
| Empty | 明确下一步，例如“请添加客户” |
| Error | 说明错误原因和修复方式 |
| Success | 短提示，不遮挡数据 |
| Disabled | 降低透明度并说明原因 |

### 8.2 表单校验

校验类型:

1. 必填。
2. 数值范围。
3. 档位区间重叠。
4. 政策有效票量大于业务量票量。
5. 成本缺失。
6. 导入字段缺失。

错误文案示例:

```text
档位区间存在重叠，请检查第 3 档和第 4 档。
政策有效票量不能大于业务量统计票量。
成本数据未填写，当前无法输出完整毛利建议。
```

### 8.3 动效

允许:

1. 数字变化轻微过渡。
2. 图表首次加载从左到右绘制。
3. Hover 出现 Tooltip。
4. 切换视角时淡入淡出。

禁止:

1. 强烈闪烁。
2. 大面积渐变动画。
3. 娱乐化庆祝动效。
4. 影响读数的持续动画。

## 9. 导出规范

### 9.1 数据导出

格式:

1. CSV。
2. XLSX。
3. JSON。

导出内容:

| 文件 | 内容 |
|---|---|
| 测算结果 | 核心指标、返利、成本、毛利、处罚 |
| 客户明细 | 客户票量、报价、成本、贡献 |
| 方案对比 | 多方案核心指标 |
| 参数模板 | 政策、网点、成本、业务量指标 |

### 9.2 图片导出

尺寸:

| 类型 | 尺寸 |
|---|---|
| 手机分享图 | 1080 x 1920 |
| 长图报告 | 1080 x 2400 或自适应高度 |
| 横向会议图 | 1920 x 1080 |
| 单卡片图 | 1200 x 900 |

图片内容:

1. 标题。
2. 测算月份。
3. 政策模板。
4. 核心结论。
5. 图表。
6. 关键数据。
7. 生成时间。
8. 免责声明。

免责声明:

```text
本结果为辅助测算，最终以管区政策和实际结算为准。
```

### 9.3 隐私处理

导出前提供选项:

1. 隐藏客户名称。
2. 隐藏具体报价。
3. 隐藏成本明细。
4. 只展示结论和区间。

## 10. 响应式与可访问性

### 10.1 断点

| 断点 | 布局 |
|---|---|
| 小于 768px | 单列 |
| 768-1199px | 双列 |
| 1200px 以上 | 三栏工作台 |
| 1600px 以上 | 最大宽度居中 |

### 10.2 移动端原则

1. 顶部只保留视角切换、月份、导出。
2. 参数区收进抽屉。
3. 客户表格改为客户卡片。
4. 图表默认展示关键区间。
5. 操作按钮高度不低于 44px。

### 10.3 可访问性

1. 文本与背景对比度满足 WCAG AA。
2. 红绿不能作为唯一信息，必须配合文字和图标。
3. 所有图表有文字摘要。
4. 表单支持键盘操作。
5. Tooltip 不承载唯一关键信息。
