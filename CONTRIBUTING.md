# 贡献说明

欢迎提交 Issue、建议和 Pull Request。

## 开发流程

1. Fork 仓库。
2. 新建分支。
3. 完成修改。
4. 运行测试。
5. 提交 Pull Request。

## 提交前检查

```bash
pnpm lint
pnpm test
pnpm build
```

在工程化版本完成前，请至少确认 Markdown 文档可读、链接有效、截图文件存在。

## 业务口径

涉及返利、处罚、成本、毛利的修改必须说明来源，并补充测试用例。

核心公式修改必须同步更新:

1. `docs/PRD-中通面单返利测算工具.md`
2. `docs/TESTING.md`
3. 对应单元测试

## 项目声明

本项目为非官方辅助测算工具，不代表中通官方政策解释。所有结果以管区最新政策和实际结算为准。
